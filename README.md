## ◈ Featured Projects

<details>
<summary><strong>RepoGuard — AI-Powered GitHub Security Scanner</strong></summary>
<br>

> Automated, AI-augmented vulnerability detection engine integrated directly into GitHub pull request workflows.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Python · FastAPI · Semgrep · Bandit · OpenAI-compatible API · GitHub Actions |
| **Scope** | PR-level automated security scanning with per-finding AI remediation |
| **Performance** | Real-time scan results surfaced inside pull request comments |
| **Security** | JWT authentication · risk scoring engine (0–100) · OWASP-mapped findings |
| **Impact** | Eliminates manual security review overhead; integrates into any Python/GitHub project |
| **Repository** | [github.com/SriHari2127](https://github.com/hariharan-a-cyber) |

</div>

RepoGuard auto-detects vulnerabilities in GitHub pull requests using Semgrep and Bandit, then layers an AI remediation engine on top to produce actionable, per-finding fix guidance. A risk scoring system (0–100) prioritises findings by severity, and GitHub Actions integration means every pull request is scanned automatically. JWT authentication secures the API surface.

<br>
</details>

---

<details>
<summary><strong>ZeroKey — Zero-Knowledge Password Manager</strong></summary>
<br>

> An Android password vault built with zero-knowledge principles — hardware-backed encryption and a phishing-resistant Autofill service.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Android · Kotlin · Jetpack Compose · Google Tink · Argon2id · Android Keystore |
| **Scope** | Zero-knowledge vault with cloud sync and credential sharing |
| **Performance** | Hardware-backed encryption via StrongBox / TEE |
| **Security** | Argon2id KDF · AES-256 envelope encryption · Frida/root detection · FLAG_SECURE · ephemeral key exchange |
| **Impact** | Phishing-resistant Autofill with domain verification; end-to-end encrypted Firestore sync |
| **Repository** | [github.com/SriHari2127](https://github.com/hariharan-a-cyber) |

</div>

ZeroKey uses Argon2id for key derivation, AES-256 envelope encryption for credentials, and Android Keystore (StrongBox/TEE) for hardware-level key protection. The Autofill service performs domain verification to defeat phishing. Cloud sync writes only ciphertext to Firestore.

<br>
</details>

---

<details>
<summary><strong>Rytm — Habit &amp; Hydration Tracker</strong></summary>
<br>

> An Android habit tracker built on Clean Architecture with a reliable alarm engine that fires even during Android Deep Sleep.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Android · Kotlin · Room · Dagger Hilt · Coroutines · Kotlin Flow |
| **Scope** | Habit and hydration tracking with reactive UI and reliable alarm delivery |
| **Performance** | `setExactAndAllowWhileIdle` alarm engine — fires during Doze mode |
| **Security** | Atomic DB transactions via Room `@Transaction` · schema-evolution-resilient backup |
| **Impact** | Full JSON backup/restore; MVVM with Kotlin Flows |
| **Repository** | [github.com/SriHari2127](https://github.com/hariharan-a-cyber) |

</div>

Rytm uses Clean Architecture, MVVM, Dagger Hilt dependency injection, and Kotlin Flows for state management. The alarm engine uses `setExactAndAllowWhileIdle` to guarantee delivery during Android Deep Sleep. Backup and restore uses JSON serialisation with atomic Room transactions.

<br>
</details>
